#1: Scrollbar Padding

  • Author: Liu Yaomin (@yaomingliu)
  • New CSS property: scrollbar-padding for a fine-grained control of the scrollbar
  • Complementary to scrollbar-gutter

Safe distance from the viewport edges

We can get the safe rectangle, where all content is visible

env(safe-area-inset-top), -bottom, -left, -right

We can control the content box

However we don't have fine-grained control of the scrollbar offsets

Declarative scrollbar padding

:root {
  --safe-y: env(safe-area-inset-top);
  --safe-x: env(safe-area-inset-right);
}

body {
  scrollbar-padding-top: --safe-y;
  scrollbar-padding-bottom: --safe-y;
  scrollbar-padding-left: --safe-x;
  scrollbar-padding-right: --safe-x;
}

#2 Maximum Fling Speed

Proposal by Qin Wen Jun @xiaolongxia

Challenge

  • Infinite-scrolling design
  • Large documents, with heavy content (e.g., media)
  • Rendering delays causing poor UX

Video here to show the challenge?

Proposal: declarative viewport behavior for fling

New <meta name="viewport"> attributes to control fling speed

<meta name="viewport" content="max-fling-speed-y=4500" >

  • max-fling-speed-x for horizontal scrolling
  • and max-fling-speed-y for vertical scrolling
  • Values to consider: pixels/second or similar

This can be done programatically using a scroll event listener. More complex and might limit the performance

#3 HTTP Headers in Speculation Rules API

Proposal by Wang Hui (@WangAoHui)

  • WebViews enable custom headers to HTTP requests for multiple purposes
  • Speculation Rules API for performance in future navigations
{
  "prefetch": [
    {
      "urls": ["next.html", "next2.html"],
      "requires": ["anonymous-client-ip-when-cross-origin"],
      "referrer_policy": "no-referrer"
    }
  ]
}

Need for HTTP Headers

  • Content Negotiation
  • Caching Controls
  • Versioning...
{
  "prerender": [
    {
      "urls": ["next.html"],
      "additional_headers": [
          {"Custom-header-id": "2"}, 
          {"Custom-header-Version":"1002"}, 
          {"Custom-header-Locale": "es-ES"}
      ]
    }
  ]
}
