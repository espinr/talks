¿Qué es el W3C?

  • World Wide Wide Consortium
  • Fundado en 1994 por el creador de la Web Tim Berners-Lee
  • Organismo neutro que desarrolla estándares para la Web
  • Misión del W3C: "Guiar la Web hacia su máximo potencial"
  • Construcción de una infraestructura para dirigir las necesidades técnicas y sociales de la Web
  • (Actividades y Grupos del W3C)

La Web Tradicional

  • Los estándares Web llevan a Internet la interoperabilidad a nivel de documento
    • HTTP provee un protocolo simple
    • Las URI ofrecen localizadores e identificadores únicos
    • HTML permite enlazar entre partes de documentos
  • Formas estándares globales para identificar y recuperar documentos...

    ... revolucionaron los negocios

XML y Servicios Web: Interoperabilidad1

The need for Web Services standards is becoming more and more important as we automate the use of so many Web services applications
Tim Berners-Lee, Director del W3C

  • Programas en la Web
    • Servicio Web: Permite interoperación entre máquinas sobre una red
      • Diferentes aplicaciones / diferentes plataformas
      • Tiene una interface bien definida
      • Tienen un comportamiento previamente definido
      • Típicamente, los envíos se realizan usando HTTP con XML
    • Estándares para los interfaces de los programas e interacciones entre estos
    • Facilitan la prestación y el uso de los servicios (y de los datos que proveen)
  • Los Servicios Web se basan en XML para estructurar la información
    • Homogeneidad para facilitar la comprensión de las máquinas
    • Diferentes plataformas / marcos de trabajo

Servicios Web

  • Tecnologías de Servicios Web
    • Protocolo Esencial
      • Transmisiones de los mensajes (SOAP 1.2)
      • Direccionamiento (Web Services Addressing)
      • Optimizaciones en la transmisión (MTOM, XOP)
    • Extensión
      • Representación de los recursos (RRSHB)
  • Descripción de Servicios Web
    • Web Services Description (WSDL 2.0)
    • Coreografías de Servicios Web (WS-CDL 1.0)

Extensiones de los Servicios Web

  • Internacionalización (WS-I18N)
    • Operaciones internacionalizadas y localizadas
    • Negociación de preferencias locales e internacionales
  • Ejemplo:
    • Soporte de múltiples preferencias internacionales y modelos de identificadores locales
    • Extracción de noticias de una base de datos en forma legible para personas

De la Web de Documentos a la Web de Datos y Servicios

  • Los recientes y futuros estándares de W3C están enfocados a
    • Extender la Web para incluir más recursos que puedan entender las máquinas
    • Activar una interoperabilidad (sintáctica y semántica) entre recursos que sabemos que son interoperables
    • Activar una interoperabilidad entre recursos que no sabemos aún si son interoperables

Web Semántica: Interoperabilidad2

  • Web Semántica: La Web de los Datos
  • Estándares para cosas, conceptos, relaciones y descripciones enlazadas en la Web
  • Facilita el entendimiento, búsqueda, reutilización, difusión, agregación y ampliación de la información
  • Exposición de los datos ocultos en los documentos, servidores y bases de datos

Evolución de la Web Tradicional...

  • Los humanos podemos entenderlo (más o menos)

... a la Web Semántica

  • Lo entendemos nosotros y las máquinas

Materializando la Web Semántica

  • Provee mecanismos específicos para las máquinas:
    • Evita la ambigüedad en la identificación (URI)
    • Expresar los datos y los metadatos (RDF)
    • Modelar ontologías (OWL)
    • Expresa reglas, permite búsquedas, establece lógica, comprobaciones, certificados de confianza (en progreso)
  • Resultado:
    • Hacer accesible lo que quieras, donde quieras, dentro de una misma empresa o entre varias

Aplicaciones de la Web Semántica

  • Áreas pioneras en el uso de la Web Semántica
    • Integración de empresas
    • Gestión documental
    • Ciencias de la vida
    • Información personal y de consumidores
  • Actualización de las organizaciones
    • Vodafone Live!, IBM, Oracle, Adobe, Nokia, HP, Fujitsu y muchas otras
  • Actualmente, los trabajos se centran en las Consultas y en las Buenas Prácticas

Reglas

  • Elemento clave para la Web Semántica
  • Facilita la integración, derivación, y transformación de los datos de diferentes fuentes de una forma distribuida, escalable, y transparente
  • Tratamiento de las reglas como datos, publicados en la Web, usando URIs
  • Las URIs sirven de enlace entre distintos sistemas de conocimiento

Por qué usar Reglas

  • Razones Técnicas
    • Evitar colisiones
    • Beneficios de la construcción de una infraestructura de red global
    • Permitir la combinación de reglas
  • Razones Sociales
    • Muchas perspectivas / Muchos backgrounds
    • Discusiones variadas
    • Minimiza los potenciales malentendidos
  • Expecialmente, en las reglas de negocio
    • Establecer/Cambiar las políticas de negocio de forma sencilla, ágil, con bajo coste
    • Al estar separadas del código, los analistas (no los programadores) pueden modificar las reglas ante ciertas circunstancias del negocio
      • Cambios en las regulaciones
      • Cambios en las políticas internas

¿Lenguaje de Reglas Estándar?

  • Menor coste y riesgo
  • Mejora la calidad mediante la especialización
    • Herramientas de autor
    • Soporte para diferentes entornos
  • El software adquirido por la organización puede integrarse de forma sencilla si está basado en estándares
  • El cliente final puede integrarlo fácilmente con sus aplicaciones

Lecciones Aprendidas

  • Data is King! Liberando los datos de las aplicaciones
  • Permitir una flexibilidad y personalización de productos / servicios de información
  • Integración de datos a la medida
  • Las cosas cambian, la planificación mejora los costes
  • Mejora de los costes incrementalmente para solucionar problemas con soluciones abiertas.

Servicios Web Semánticos: Operabilidad3

  • Intraestructura para especificar Semántica en los Servicios Web
    • Ontologías para describir los Servicios Web
    • Razonamiento basado en la semántica
      • Composición
      • Descubrimiento
      • Elección
      • Control de versiones

Semántica en las Descripciones de los Servicios Web

  • Los Servicios Web sólo definían a nivel sintáctico sus interfaces
  • Descripción semántica de forma que las máquinas puedan entender
    • Lo que pretende realizar un servicio
    • Los tipos de datos que serán manejados
  • Descripciones de alto nivel que permitirían descubrir y componer los servicios de forma sencilla

La Importancia de la Semántica

  • Ejemplos de uso:
    • Conexión de dos servicios asegurándose que entienden el mismo idioma
      • Paso de una fecha entre empresas en distintos paises
    • Mezcla de datos de distintas fuentes
      • Title, Titre y Título
    • De una gran lista de servicios, seleccionar exactamente el que el hace lo que el usuario demanda
    • Recuperaciones automáticas ante fallos

Evolución lógica

  • Servicios Web = Programas
    • Transmisión (SOAP)
    • Descripción (WSDL)
  • Web Semántica = Datos
    • Bases de conocimiento
    • Ontologías (OWL)
    • Reglas
    • Bases de Datos (RDF, SPARQL)
