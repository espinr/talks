2. Security

  • Seguridad Web es uno de los grandes problemas
  • Problemas para la Web:
    • Pérdida/robo de datos, amenazas de malware, vigilancia
    • Baja seguridad conlleva pérdida de confianza e incrementa costes
  • La Open Web Platform cada vez más poderosa
  • Millones de dispositivos conectados en la (WoT)
  • Necesaria la confianza para comunicaciones personales y en negocios
  • Proyectos: Web Crypto, WebAppSec, Web Authentication

3. WebRTC

  • La OWP permite aplicaciones multiplataforma, ricas e interactivas
  • Necesidad de conversaciones en tiempo real
  • WebRTC extiende los servicios de comunicación en tiempo real para cualquier dispositivo sobre cualquier red

4. Web Payments

  • e-Commerce potential: 1.5 trillion USD annually
  • Evolución del e-Commerce hacia una evolución basada en la Web
  • Los clientes son el centro del e-Commerce
  • Los clientes combinan experiencias in-store (móvil) y on-line
  • Flexibilidad y seguridad en los métodos de pago en los distintos canales
  • La infraestructura de pagos y del e-Commerce será basada en la Web
  • La Web conectará a las personas con el potencial económico global

5. Apoyo a la industria

  • Cada vez es mayor el impacto de la Open Web Platform
  • Más sectores están construyendo su negocio sobre la Web
  • El W3C necesita comprender a los distintos sectores
  • La industria ya es un foco del W3C
  • Programa: Industry Champion (∗)

6. WoT

Uniendo los silos, habilitando un mercado de "cosas"

  • La Web de las Cosas es una capa de abstracción e interoperabilidad sobre IoT
  • Transición del costoso desarrollo de software monolítico a los mercados de apps abiertos
  • Dispositivos conectados: 38.500m en 2020 (from 13.400m en 2015) (Juniper)

8. Aplicaciones Web Distribuidas

  • Usuarios con varios dispositivos (y pantallas)
  • Varias pantallas para la convergencia entre Web y TV
  • La Web debe habilitar protocolos para comunicación entre los dispositivos "clásicos" (smartphones, laptops, etc.) y los "nuevos" (Google Cast, Apple Airway, Netflix' DIAL)
  • Primer paso: Presentation API

9. Sincronización entre dispositivos

  • Sincronización de datos y recursos entre distintos dispositivos
    • Usuario viendo la TV y interactuando con una tablet
  • Distintos usuarios haciendo la misma tarea
    • Usuarios editando documentos on-line o viendo vídeo sincronizado
  • Distintos dispositivos combinados para la misma tarea
    • Video-wall, música distribuida, escenarios de señalización digital…

Polémica: ¿DRM y W3C?

  • Vídeos nativos en HTML5
  • Digital Rights Management para restringir el uso de obras con copyright
  • La FSF y otros no admiten DRM en ningún caso en la Web
  • (Feb 2012) Miembros W3C proponen Encrypted Media Extensions (EME) para extender HTMLMediaElement
  • EME permitirá la comunicación entre los navegadores Web y los agentes de gestión de DRM

Más info sobre esto: https://www.w3.org/2016/03/EME-factsheet.html
