Mecanismos básicos en Web Semántica

  • RDF (Infraestructura para Definición de Recursos)
    • Modelo simple para la representación de los metadatos

    • Definen términos y relaciones
    • Lenguajes de Ontologías:
      • RDF-Schema
      • OWL
  • Reglas de búsqueda y consulta

RDF (Resource Description Framework)

  • Permite definir información sobre cualquier dominio
  • Todo lo que se describen son recursos (identificados por URIs)
  • Es un gráfico
  • Definido como una tripleta (s,p,o)
    • Sujeto - Recurso (http://www.w3c.es/Personal/Martin)
    • Predicado - Propiedad (creador)
    • Objeto - Literal ("Martín Álvarez")
  • <sujeto> TIENE <predicado> CUYO VALOR ES <objeto>
    • http://www.w3c.es/Personal/Martin TIENE un creator cuyo valor es Martín Álvarez

Ejemplo de RDF

  • Extensibilidad incremental
  • Relacionarse con otros recursos
  • "Mezclar" definiciones
    • Nodos sujeto y objeto con el mismo URI
  • Proporcionar múltiples presentaciones aprovechando el trabajo de otros

Por qué usar ontologías

  • Necesitamos definir vocabularios (términos) para usarlos en estas sentencias
  • Descripción formal de un dominio
    • Definición de clases, propiedades y relaciones entre ellas
  • Interoperabilidad semántica
    • Integración entre metadatos de distintos vocabularios/esquemas
    • Facilita y ofrece mayor precisión ante búsquedas de información
      • Portales (reglas de categorización para los recursos)
      • Colecciones multimedia (medios no textuales)
      • Sitios web corporativos (organización taxonómica de documentos)
    • Relaciona recursos, servicios, dispositivos, bases de datos a través de la asociación de las estructuras del conocimiento y su combinación.

RDF Schema

  • RDF Schema es un Lenguaje de Descripción de Vocabularios
  • Modelo de datos para RDF
  • Permite enriquecer las descripciones semánticas (procesables por las máquinas)
    • Clases / subclases
    • Relaciones entre las clases y propiedades
    • Descripciones para los humanos (etiquetas, descripciones, etc.)
  • Facilita la combinación de sentencias RDF, a través de los URIs
  • Ofrece la base para poder realizar razonamientos o deducciones sobre nueva información
  • Básicamente permite definir clases y propiedades e indica cómo se pueden usar en conjunto

Limitaciones de RDF Schema

  • No indica la cardinalidad
    • Un libro: ¿Puede tener más de un autor? ¿Puede no tener ninguno?
  • Tipos de datos
  • Restricciones de rangos simples para las propiedades
  • No se pueden derivar tipos por unión o intersección
  • Necesidad de relaciones de clases o propiedades más ricas
    • Características de las propiedades
  • ...
  • Surgen extensiones DAML+OIL

OWL

  • Lenguaje de Ontologías Web (Recomendación Febrero 2004)
  • Se basa en RDF Schema
  • Añade más vocabulario para describir las clases y las propiedades
  • Los lenguajes anteriores estaban destinados a comunidades específicas
    • Comercio Electrónico
    • Entornos científicos
  • OWL se adapta a la arquitectura de la WWW y a la Web Semántica
  • Los sistemas que usan DAML, OIL, DAML+OIL están migrando a OWL
    • Muchas herramientas que lo realizan sencillamente

Necesidad de Ontologías en OWL

  • RDF Schema es útil pero no lo podemos utilizar para resolver ciertas tareas
  • Aplicaciones complejas pueden necesitar más funcionalidad
    • ¿Puede un programa razonar acerca de ciertos términos?
      • p.e., "si A es menor que B y B es menor que C, ¿es A menor que C?"
      • Es obvio para los humanos pero no para las máquinas...
      • Los programas deberían poder deducir estas sentencias
    • Si alguien define un conjunto de términos: ¿son estos idénticos?
      • Obviamente, puede aplicarse en el contexto de internacionalización
    • Construir clases, no solo nombrarlas
    • Restringir el rango de las propiedades
    • etc.

¿Por qué "OWL" y no "WOL"

  • Algunas leyendas urbanas...
    • p.e., referencia a Owl de Winnie the Pooh, que deletreaba mal su nombre como "WOL"
  • Una referencia a un proyecto de IA de mediados de los 70 de Bill Martin llamado "One World Language"
    • Un intento de un lenguaje de Representación del Conocimiento y su ontología asociada, que intentaba ser un lenguaje universal para codificar significados en las máquinas
  • "Why not be inconsistent in at least one aspect of a language which is all about consistency" (Guus Schreiber)

Clases en OWL

  • En RDF Schema ya podían definirse subclases, pero con OWL:
  • Pueden construirse clases de otras existentes previamente
    • Enumerarlas en su contenido
    • A través de intersección, unión, complementarios
    • A través de restricciones apropiadas
  • OWL introduce Class y Thing para diferenciar las clases y los individuos

Operandos de conjuntos sobre clases

  • Unión de conjuntos mediante unionOf
  • Otras posibilidades: intersectionOf y complementOf

Clases disjuntas

  • owl:disjointWith
  • Una instancia de una clase no puede pertenecer simultáneamente a otra clase
  • p.e., ontología de comida: Definición de Carne

  • Nota: Sólo se realiza el aserto sobre la clase Carne
    • No se conoce si Pescado es disjunto a Marisco

Restricciones de propiedades

  • Las clases pueden crearse mediante la restricción del comportamiento en esa clase
  • Limitación específica del rango
  • Las restricciones pueden ser:
    • De valores
      • ...todos los valores deben ser de cierta clase
      • ...al menos un valor debe ser de cierta clase
    • De cardinalidad (p.e., ¿cuántas veces puede usarse una propiedad en una instancia?)
      • cardinalidad máxima
      • cardinalidad mínima
      • cardinalidad exacta

Ejemplo de restricción de propiedades

  • Un Zumo es un LíquidoPotable que está hecho al menos por 1 pieza de fruta

  • Otras restricciones: allValuesFrom, someValuesFrom y hasValue

Caracterización de propiedades

  • En RDF Schema las propiedades se restringen mediante rango ( range) y dominio (domain)
  • En OWL, se puede caracterizar el comportamiento
    • Propiedad transitiva, simétrica, funcional, etc.
  • OWL distingue las propiedades de datos
    • DatatypeProperty se refiere a rangos sobre literales de RDF o de tipos simples en XML Schema
  • Muy útil para aplicaciones basadas en ontologías

Necesidades adicionales

  • Las ontologías pueden llegar a ser muy grandes
    • Necesitan cuidados especiales
    • Pueden componerse en varios módulos
    • Pueden provenir de varios lugares y deben ser integrados
  • Las ontologías están en la web
    • Las aplicaciones usan varias, diferentes ontologías
    • Utilizar la misma ontología pero en distintos idiomas
    • Equivalencias y relaciones entre términos

Equivalencias

  • Entre clases y propiedades
    • equivalentClass y equivalentProperty
  • Entre individuos
    • sameAs
    • differentFrom, AllDifferent

Control de versiones y anotaciones

  • La clase owl:Ontology permite "documentar" y asociarla con otras ontologías
    • owl:imports
    • owl:versionInfo, owl:priorVersion
    • owl:backwardCompatibleWith, owl:incompatibleWith
    • rdfs:label, rdfs:comment
  • Control de versiones obsoletas
    • owl:DeprecatedClass y owl:DeprecatedProperty

Lenguajes OWL

  • 3 Sublenguajes con expresividad creciente
  • OWL Lite
    • Para usuarios que necesitan principalmente una clasificación jerarquizada y restricciones sencillas
    • Sólo permite expresar cardinalidad para valores 0 o 1
  • OWL DL (Description Logics)
    • Máxima expresividad
    • Manteniendo la computación completa (se garantiza la computación para todas las conclusiones) y decidible (tiempo finito)
    • Admite todas las construcciones del lenguaje OWL pero con ciertas restricciones
  • OWL Full
    • Máxima expresividad y libertad sintáctica de RDF
    • No ofrece garantías de computación
    • Es improbable que un software pueda soportar el razonamiento completo para cada característica de OWL Full

¿Usar ontologías OWL?

  • Una aplicación basada completamente en ontologías es un sistema muy complejo
    • Muy tediosa la implementación
    • Requiere muchos recursos el funcionamiento
  • No todas las aplicaciones necesitan ontologías con OWL
  • Puede llegar a complicar demasiado el sistema

SKOS Core

  • Simple Knowledge Organisation System(s)
  • Objetivo:

    Infrastructura simple, flexible, extensible para representar (de forma que las máquinas lo entiendan)

    • Tesauros
    • Esquemas de clasificación
    • Taxonomías
    • Listas de encabezamientos de materia
    • Otros tipos de "vocabulario controlado"

La base de SKOS Core

  • SKOS Core es una aplicación RDF
    • Recordando...
    • RDF es un mecanismo de representación formal de cosas, sus propiedades, relaciones entre ellas, y categorías (clases)
  • Clases RDF y Propiedades RDFS
  • Es usado para describir Conceptos y Esquemas de Conceptos

Concepto

  • Definiciones (RAE):
    • Idea que concibe o forma el entendimiento
    • Pensamiento expresado con palabras
  • Por ejemplo, una unidad de significado
  • Es la unidad que compone los Esquemas de Conceptos

Representación del concepto

  • skos:Concept es una clase que indica que un recurso es un recurso conceptual
  • Elementos:

    • Recursos
    • Propiedades
    • Clases
    • Sentencias

Esquema de Conceptos

  • Un conjunto de conceptos,
    • Opcionalmente incluye sentencias sobre las relaciones semánticas entre los conceptos
  • P.e.
    • Tesauro
    • Esquema de clasificación
    • Taxonomía
    • Terminología
    • Epígrafes
    • ... otros vocabularios controlados

Relaciones con las ontologías RDFS/OWL

  • Diferencias entre SKOS Core y otras aplicaciones RDF como FOAF
    • En términos de lo que permiten modelar
  • SKOS Core permite modelar conceptos / FOAF permite modelar objetos (personas, lugares, etc.)

Otras características

  • Etiquetado multilingüe
  • Etiquetado simbólico
  • Anotaciones / Documentación
  • Ofrece soporte para colecciones
  • Esquemas conceptuales
  • Extensible
  • Por supuesto que se pueden usar propiedades y clases de otros vocabularios RDF
    • Téminos DCMI (Dublin Core Metadata Initiative), FOAF (Friend Of A Friend), DOAP (Description Of A Project), etc.
